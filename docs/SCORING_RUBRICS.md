# Scoring Rubrics (채점 기준)

모든 점수는 1-5 정수 스케일. 가중 합산으로 종합 점수 산출.

## 1. 구현정확도 (Implementation Accuracy) — 가중치 30%

생성된 코드가 실제로 올바르게 동작하는지 평가합니다.

| 점수 | 기준 | 예시 |
|------|------|------|
| **5** | 모든 테스트 케이스 통과, 엣지케이스 완벽 처리 | 빈 입력, 경계값, 대규모 입력 모두 처리 |
| **4** | 핵심 로직 정확, 마이너 엣지케이스 1개 누락 | 정상 입력은 완벽하지만 빈 리스트 미처리 |
| **3** | 핵심 로직 대체로 정확, 엣지케이스 2개+ 누락 | 기본 동작은 하지만 여러 경계 조건 실패 |
| **2** | 부분 구현, 근본적 로직 오류 존재 | 알고리즘 접근은 맞지만 핵심 로직에 오류 |
| **1** | 컴파일/실행 불가 또는 완전히 잘못된 접근 | SyntaxError 또는 문제를 완전히 오해 |

**자동 채점 매핑**: pass_rate → 100%=5, 75%=4, 50%=3, 25%=2, 0%=1

## 2. 의도파악 (Intent Comprehension) — 가중치 25%

모델이 프롬프트의 의도를 얼마나 정확히 이해했는지 평가합니다.

| 점수 | 기준 | 예시 |
|------|------|------|
| **5** | 모든 요구사항과 제약조건을 완벽히 이해 | "에러 핸들링 포함" 요구에 try-except + 커스텀 에러 구현 |
| **4** | 주요 의도 파악, 부차적 요구사항 1개 누락 | 기능은 맞지만 "타입 힌트 포함" 요구 누락 |
| **3** | 주요 의도 파악했으나 뉘앙스 놓침 | 함수를 만들었지만 클래스로 요구한 것을 무시 |
| **2** | 부분적 이해, 상당한 오해 존재 | 정렬을 요구했는데 검색을 구현 |
| **1** | 프롬프트를 완전히 오해 | 전혀 다른 기능을 구현 |

## 3. 할루시네이션 (Hallucination) — 가중치 20%

존재하지 않는 API, 함수, 라이브러리를 지어냈는지 평가합니다.

| 점수 | 기준 | 예시 |
|------|------|------|
| **5** | 할루시네이션 없음, 모든 API/함수 실재 | 모든 import와 함수 호출이 실제 존재 |
| **4** | 경미한 부정확 1개 | `requests.get()` 파라미터명 오류 (timeout → time) |
| **3** | 경미한 부정확 2-3개 또는 가짜 API 1개 | `pandas.DataFrame.to_excel(auto_format=True)` (존재하지 않는 파라미터) |
| **2** | 다수의 가짜 함수/API | 여러 존재하지 않는 라이브러리 함수 사용 |
| **1** | 전반적 할루시네이션 | `import smartutils` 같은 존재하지 않는 패키지 다수 |

## 4. 코드버그 발생도 (Code Bug Rate) — 가중치 15%

생성된 코드에 포함된 버그의 수와 심각도를 평가합니다.

| 점수 | 기준 | 예시 |
|------|------|------|
| **5** | 버그 없음 | 깨끗하고 정확한 코드 |
| **4** | 마이너 버그 1개 | Off-by-one 에러, 포맷팅 이슈 |
| **3** | 마이너 버그 2-3개 또는 심각한 버그 1개 | 변수 미초기화, 무한 루프 가능성 |
| **2** | 다수의 심각한 버그 | 메모리 누수 + 경쟁 조건 + 미처리 예외 |
| **1** | 코드 작동 불가 | 버그가 너무 많아 실행 자체가 불가능 |

## 5. 누락/아웃데이트 빈번도 (Omission/Outdated) — 가중치 10%

요구사항 누락이나 deprecated 패턴 사용 여부를 평가합니다.

| 점수 | 기준 | 예시 |
|------|------|------|
| **5** | 모든 요구사항 충족, 최신 패턴 사용 | Python 3.11+ 문법, 최신 라이브러리 API |
| **4** | 경미한 누락 1개 또는 약간 구식 패턴 | `format()` 대신 f-string 미사용 |
| **3** | 누락 2개+ 또는 deprecated API 사용 | `asyncio.get_event_loop()` (deprecated) |
| **2** | 주요 요구사항 누락 또는 상당히 구식 | Python 2 스타일 코드 |
| **1** | 대부분 요구사항 누락, 완전히 구식 접근 | 핵심 기능 대부분 빠짐 |

## 종합 점수 (Composite Score)

```
composite = (impl_accuracy × 0.30) + (intent × 0.25) + (hallucination × 0.20) + (bugs × 0.15) + (omission × 0.10)
```

| 종합 점수 | 해석 |
|-----------|------|
| 4.5 - 5.0 | Excellent — 거의 완벽한 응답 |
| 3.5 - 4.4 | Good — 실용적으로 사용 가능 |
| 2.5 - 3.4 | Fair — 수정 필요하지만 방향은 맞음 |
| 1.5 - 2.4 | Poor — 상당한 수정 필요 |
| 1.0 - 1.4 | Terrible — 사실상 사용 불가 |
